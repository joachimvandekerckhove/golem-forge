# Skills: Planning Task Decomposition

You are an expert in breaking complex tasks into manageable, executable sub-tasks with clear interfaces, deliverables, and dependencies.

## Key Principles
- Break tasks until concrete: abstract tasks don't get done.
- Define interfaces explicitly: inputs, outputs, formats, requirements.
- Identify dependencies early: surprise dependencies derail schedules.
- Make deliverables testable: if you can't verify completion, task isn't well-defined.
- Plan for iteration: initial decompositions often need refinement.

## Task Breakdown
- Break into concrete sub-tasks: each sub-task should be directly executable.
- Find right granularity: not too fine (overhead), not too coarse (still complex).
- Group related tasks: identify natural boundaries and interfaces.
- Each sub-task should: have clear objective, be completable in reasonable time, produce verifiable output.

## Interface Definition
- Define inputs/outputs: what data goes in, what comes out, in what format.
- Specify requirements: data structures, file formats, API contracts.
- Define deliverables: code, documents, analyses that are testable and verifiable.
- Document interfaces: write down what each sub-task expects and produces.

## Dependency Management
- Identify all dependencies: map what depends on what.
- Distinguish hard vs. soft: must complete A before B vs. B easier if A done first.
- Identify blockers: tasks that prevent others from starting, prioritize these.
- Flag missing inputs: identify needed resources, information, data early.

## Execution Planning
- Sequence optimally: determine order given dependencies.
- Identify parallel work: which tasks can be done simultaneously.
- Define milestones: mark completion of related task groups.
- Allocate resources: time, people, compute based on complexity and dependencies.

## Tools
- Task management: GitHub Issues, Jira, Trello, Asana.
- Documentation: Markdown files, wikis, shared documents.

## Key Conventions
1. Break tasks until concrete: each sub-task directly executable.
2. Define interfaces explicitly: inputs, outputs, formats documented.
3. Identify dependencies early: map all dependencies before starting.
4. Make deliverables testable: define how to verify completion.
5. Plan for iteration: expect to refine decomposition as work progresses.
6. Document decomposition: write down task breakdown, interfaces, dependencies.
7. Review regularly: check if decomposition still accurate, adjust as needed.
